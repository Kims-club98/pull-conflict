## 충돌사례

### Case 1
원격레포와 로컬 레포의 커밋 이력이 같은 상황에서 로컬에서 working directory에서의 수정 이후 git pull을 한다면..?

```bash
# pull = fetch + merge
# fetch는 성공인데, merge 가 실패한 경우
git pull origin main
```

충돌이 아니므로 그냥 git push 를 수행하면 된다.

### Case 2
원격레포에서 commit 이력을 추가하였을 때 충돌이 없다.
git pull origin main을 통해서 commit 이력을 내려받은 후 병합의 일종으로 볼 수 있으나, 변경사항에 대한 head만 이동하므로 fast forwasrd에 해당한다.

### Case 3
원격에 Commit 이력이 추가 발생시킴
또 로컬에서도 별도의 Commit 이력이 발생한다.
단. 같은 파일에 대한 수정이 아닐 경우 -> 충돌이 발생하지 않는다(같은 파일에 대한 수정 => 충돌발생)
git pull시 문제 없이 실행되고 단 fas forword가 아닌 merge commit 이 발생한다.
이 경우에는 다시 pull이 필요하다.
pull = fetch + merge (pull은 fetch와 merge가 별도로 발생하며, 둘 중 하나가 발생하지 않을 경우도 있음)

### Case 4 - 가장 빈번한 경우
원격에 commit이력 발생, 동시에 로컬을 별도로 commit이 발생함.
해결방안
  git pull origin main 하면 각각의 commit 아이디가 있는 경우이므로
  (참고로 case#5의 경우는 한쪽만 commit이 발생한 경우이다.)
  vsCode로 GUI이미지로 화면을 제공해준다.
  틀린 부분에 대하여 병합 파일을 생성해준다.
  
 <병합 파일을 적절하게 수정 후>
 ```sh
 git add .
 git commit -m "commit 메시지"
 git push origin main
 ```

 방법2
 git fetch -> git diff -> git merge orgin/main
 충돌 해결 후 다시
 ```sh
git push origin main
 ```

 ### Case5
 원격에 커밋 이력 추가 발생 시 && working directory 또는 staging에서 같은 작업 시 git pull origin main -> 충돌발생
 로컬에서는 커밋 아이디가 없고, 원격에서는 커밋 아이디가 있어서 Case 4와 다르게 GUI로 화면제공이 되지 않는다.

 error: Your local changes to the following files: would be overwritten by merge

 즉
 로컬에 commit이력이 존재하지 않아서 merge 할 수 있는 병합파일이 생성되지 않는 경우

 방법1.
 로컬에서 커밋을 만든 후 git merge origin/main 충돌 제거 후
 ```sh
 git add .
 git commit -m "메시지"
 git push orgin main
```

방법2
기존 수정본을 backup을 받아두고 로컬 작업을 취소한다.
-> git pull orgin main
-> backup파일을 참고하여 작업사항을 반영
